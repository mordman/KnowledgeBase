# 🔌 UID в оборудовании: идентификация USB-устройств

В контексте «железа», и особенно USB-устройств, термин **UID (Unique Identifier)** не является единым стандартом. Вместо него используется **набор идентификаторов**, каждый из которых решает свою задачу.

---

## 🔹 Иерархия идентификаторов USB-устройства

### 1. **VID / PID** (Vendor ID / Product ID)
| Поле | Размер | Описание | Можно ли изменить? |
|------|--------|----------|-------------------|
| `idVendor` | 16 бит | Уникальный код производителя, выдаётся USB-IF [[3]] | ❌ Нет (запрограммирован в прошивке) |
| `idProduct` | 16 бит | Код модели устройства, задаётся производителем [[3]] | ❌ Нет (только перепрошивкой) |

📌 **Пример**: `VID:0x046D` (Logitech), `PID:0xC52B` (Unifying Receiver)

> ⚠️ VID/PID идентифицируют **тип устройства**, а не конкретный экземпляр. Два одинаковых флеш-накопителя будут иметь одинаковые VID/PID.

---

### 2. **iSerialNumber** (Серийный номер устройства)
| Поле | Тип | Описание | Можно ли изменить? |
|------|-----|----------|-------------------|
| `iSerialNumber` | Индекс строкового дескриптора | Уникальный серийный номер, заданный производителем [[9]][[17]] | ⚠️ Теоретически да, но сложно |

🔍 **Важные нюансы**:
- Поле **необязательно**: многие дешёвые устройства не имеют серийного номера [[9]]
- Хранится в **строковом дескрипторе** USB, а не в постоянном идентификаторе чипа [[17]]
- В Windows отображается в:  
  `Диспетчер устройств → Свойства устройства → Сведения → ИД оборудования` или через утилиты типа **USBDeview** [[13]]

#### 🔧 Как изменить серийный номер?
| Способ | Реализуемость | Риски |
|--------|---------------|-------|
| **Официальная утилита производителя** (например, Cypress USB-Serial Configuration Utility) | ✅ Возможно для некоторых чипов [[18]] | Низкие |
| **Перепрошивка контроллера** (через производительские инструменты) | ⚠️ Сложно, требует знаний и доступа к документации | Высокие: можно «окирпичить» устройство |
| **Magic Card / поддельные контроллеры** | ⚠️ Только для специфических чипов (не для массовых флешек) [[26]] | Нарушение гарантии, юридические риски |
| **Программная подмена на уровне ОС** (udev в Linux, реестр Windows) | ✅ Да, но только для отображения, не меняет реальный дескриптор | Низкие, но не решает задачу аппаратной идентификации |

> 💡 **Вывод**: Штатно изменить `iSerialNumber` **нельзя**. Это защита от клонирования и подделки [[20]][[22]].

---

### 3. **Container ID / UUID** (USB 3.0+)
| Поле | Стандарт | Описание |
|------|----------|----------|
| `Container ID` | USB 3.0 Specification | 128-битный UUID, уникальный для физического устройства [[6]] |

🔹 Предназначен для привязки драйверов и настроек к конкретному экземпляру устройства в ОС.  
🔹 Генерируется при первом подключении или зашивается производителем.  
🔹 **Не редактируется пользователем**.

---

### 4. **Volume Serial Number** (Серийный номер тома)
❗ **Не путать** с `iSerialNumber`!

| Параметр | Где хранится | Что идентифицирует |
|----------|--------------|-------------------|
| **Device Serial** (`iSerialNumber`) | В прошивке контроллера USB | Само физическое устройство [[13]] |
| **Volume Serial** | В загрузочной записи файловой системы (FAT/NTFS) | Конкретный раздел/том на устройстве [[13]] |

🔹 Volume Serial генерируется при форматировании и **легко меняется**:
```cmd
:: Windows (через форматирование)
format X: /FS:NTFS /V:MyLabel

:: Linux (для FAT32)
sudo fatlabel /dev/sdX1 NEW_LABEL
sudo mkfs.vfat -n NEW_LABEL /dev/sdX1  # с потерей данных
```
⚠️ Это **не меняет** аппаратный идентификатор устройства — только метку тома.

## 🔹 Как посмотреть идентификаторы на практике

### ▶ В Linux:
```bash
# Полная информация об устройстве
lsusb -v | grep -E "idVendor|idProduct|iSerial"

# Через sysfs (более надёжно)
udevadm info --query=all --name=/dev/sdX | grep -E "ID_SERIAL|ID_USB_DRIVER"

# Уникальный путь устройства (стабильный для udev-правил)
udevadm info --query=path --name=/dev/sdX
```

### ▶ В Windows:
1. Откройте **Диспетчер устройств** → найдите устройство → Свойства → вкладка **Сведения**
2. Выберите в выпадающем списке:
   - `ИД оборудования` → покажет `USB\VID_xxxx&PID_xxxx...`
   - `Путь к экземпляру устройства` → содержит серийный номер, если есть [[15]]
3. Или используйте **USBDeview** (NirSoft) для удобного просмотра всех полей [[13]]

### ▶ В коде (WebUSB API, экспериментально):
```javascript
// Требует HTTPS и разрешения пользователя
navigator.usb.requestDevice({ filters: [] })
  .then(device => {
    console.log('Serial:', device.serialNumber); // [[8]]
    console.log('Vendor:', device.vendorId.toString(16));
    console.log('Product:', device.productId.toString(16));
  });
```
⚠️ Поддерживается только в Chromium-браузерах, в защищённом контексте [[8]].

## 🔹 Можно ли «подменить» UID USB-устройства?

| Цель | Реализация | Надёжность |
|------|------------|------------|
| **Скрыть устройство от учёта** | Программная фильтрация на уровне драйвера/udev | ⚠️ Обходится продвинутыми системами мониторинга |
| **Клонировать устройство** | Перепрошивка контроллера + копирование дескрипторов | ❌ Сложно, требует доступа к исходному чипу и инструментам; часто блокируется однократной записью OTP [[26]] |
| **Эмулировать устройство** | Использование программируемых платформ (Raspberry Pi Pico, Arduino с USB-стек) | ✅ Да, но это будет новое устройство с другими дескрипторами |
| **Изменить отображаемый серийник в ОС** | Правка реестра Windows / udev-правила в Linux | ⚠️ Только локально, не влияет на другие системы |

> 🔐 **Безопасность**: Производители всё чаще используют **аппаратно-неизменяемые идентификаторы** и криптографическую аутентификацию (например, в токенах YubiKey), чтобы предотвратить подмену [[5]].

## ✅ Что делать, если вам нужно уникально идентифицировать USB-устройство?

1. **Проверьте наличие `iSerialNumber`** — если есть, используйте его как основной UID.
2. **Если серийника нет** — скомбинируйте:
   ```
   Хеш(VID + PID + Путь_в_sysfs + Время_подключения)
   ```
   Это даст стабильный идентификатор в рамках одной системы.
3. **Для кросс-платформенной идентификации** — используйте `Container ID` (если устройство поддерживает USB 3.0+) или храните собственный токен в памяти устройства (если есть доступ на запись).
4. **Для защиты от подмены** — не полагайтесь только на серийный номер; добавьте криптографическую проверку (подпись, challenge-response).
