- **Назначение**: Создание нового ресурса на сервере.
- **Использование**:
  - Отправка данных для создания объекта (например, добавление нового пользователя, публикации, заказа).
  - Обычно не идемпотентен: повторный запрос может создать дубликат.
- **Пример**:
  ```http
  POST /users
  {
    "name": "Dima",
    "email": "dima@example.com"
  }
  ```

---

### **2. GET**
- **Назначение**: Получение ресурса с сервера.
- **Использование**:
  - Запрос данных (например, получение списка пользователей или информации о конкретном пользователе).
  - Идемпотентен: повторный запрос не изменяет состояние сервера.
- **Пример**:
  ```http
  GET /users/1
  ```

---

### **3. PUT**
- **Назначение**: Обновление существующего ресурса **или** его создание, если ресурса нет.
- **Использование**:
  - Полная замена ресурса (например, обновление всех полей пользователя).
  - Идемпотентен: повторный запрос не изменяет результат.
- **Пример**:
  ```http
  PUT /users/1
  {
    "name": "Dima Morozov",
    "email": "dima.morozov@example.com"
  }
  ```

---

### **4. DELETE**
- **Назначение**: Удаление ресурса с сервера.
- **Использование**:
  - Удаление объекта (например, пользователя, публикации).
  - Может быть идемпотентен (если повторный запрос не вызывает ошибку, если ресурс уже удалён).
- **Пример**:
  ```http
  DELETE /users/1
  ```

---

### **5. PATCH**
- **Назначение**: Частичное обновление ресурса.
- **Использование**:
  - Изменение только определённых полей (например, обновление только email пользователя).
  - Не идемпотентен: результат зависит от текущего состояния ресурса.
- **Пример**:
  ```http
  PATCH /users/1
  {
    "email": "new.email@example.com"
  }
  ```

---

### **6. HEAD**
- **Назначение**: Получение метаданных ресурса (без тела ответа).
- **Использование**:
  - Проверка существования ресурса или получение заголовков (например, для кеширования).

---

### **7. OPTIONS**
- **Назначение**: Получение информации о поддерживаемых методах для ресурса.
- **Использование**:
  - Определение, какие HTTP-методы разрешены для конкретного URL.

---

### **Ключевые отличия**
| Метод  | Идемпотентность | Безопасность | Назначение                     |
|--------|-----------------|--------------|---------------------------------|
| POST   | Нет             | Нет          | Создание ресурса               |
| GET    | Да              | Да           | Получение ресурса              |
| PUT    | Да              | Нет          | Полное обновление/создание     |
| DELETE | Да*             | Нет          | Удаление ресурса               |
| PATCH  | Нет             | Нет          | Частичное обновление           |

*DELETE может быть идемпотентен, если сервер не возвращает ошибку при повторном запросе.
