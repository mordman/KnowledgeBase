# 📋 Описание приложения "Оптимальная Диета"

## 🎯 Назначение

Приложение для автоматического составления оптимального рациона питания на один день с учётом:
- Калорийности
- Баланса белков, жиров, углеводов (БЖУ)
- Бюджета на продукты
- Количества приёмов пищи
- Количества блюд в каждом приёме

---

## ✨ Основные возможности

| Функция | Описание |
|---------|----------|
| 🎲 **Автоподбор диеты** | Алгоритм подбирает комбинации блюд, максимально близкие к целевым показателям |
| 📊 **Гибкие ограничения** | Настройка допустимых отклонений в процентах для каждого параметра |
| 🍽 **Структура питания** | Настройка количества приёмов пищи и блюд в каждом приёме |
| 📝 **Рецепты блюд** | Каждое блюдо содержит подробный рецепт приготовления |
| ⚖️ **Вес порций** | Учёт веса каждого блюда и общего веса за день |
| 💰 **Контроль бюджета** | Расчёт общей стоимости дневного рациона |
| 📥 **Загрузка JSON** | Импорт списка блюд из JSON-файла |
| 💾 **Экспорт результата** | Сохранение готового плана питания в JSON-файл |

## 🖥 Интерфейс приложения

### Левая панель — Параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| Ккал | Целевая калорийность дня | 2000 |
| Белки (г) | Целевое количество белков | 100 |
| Жиры (г) | Целевое количество жиров | 80 |
| Углеводы (г) | Целевое количество углеводов | 250 |
| Цена (руб) | Максимальный бюджет на день | 500 |
| % отклонения | Допустимое отклонение от цели | 10% |
| Приёмов пищи | Количество раз в день | 3 |
| Блюд (мин/макс) | Количество блюд в приёме | 1-2 |

### Правая панель — Данные

- **Поле ввода JSON** — список доступных блюд
- **Кнопка "Загрузить пример"** — заполняет поле тестовыми данными
- **Кнопка "Проверить JSON"** — валидация формата данных
- **Поле результата** — JSON с готовым планом питания
- **Кнопка "Сохранить в файл"** — экспорт результата

## 📊 Формат входных данных (JSON блюд)

```json
[
  {
    "name": "Овсянка на воде",
    "calories": 350,
    "proteins": 12,
    "fats": 6,
    "carbs": 60,
    "price": 50,
    "weight": 250,
    "recipe": "1. Залить овсяные хлопья водой...\n2. Варить 10 минут..."
  }
]
```

### Обязательные поля
| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Название блюда |
| `calories` | number | Калорийность (ккал) |

### Необязательные поля (по умолчанию = 0)
| Поле | Тип | Описание |
|------|-----|----------|
| `proteins` | number | Белки (г) |
| `fats` | number | Жиры (г) |
| `carbs` | number | Углеводы (г) |
| `price` | number | Цена (руб) |
| `weight` | number | Вес порции (г) |
| `recipe` | string | Рецепт приготовления |

## 🔧 Алгоритм работы

```
┌──────────────────────────────────────────────────────────────┐
│  1. Загрузка списка блюд из JSON                             │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  2. Генерация всех вариантов ОДНОГО приёма пищи              │
│     (комбинации блюд от min до max)                          │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  3. Фильтрация и сортировка вариантов                        │
│     (близость к идеалу = Цель / Кол-во приёмов)              │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  4. Сборка полного дня (Монте-Карло)                         │
│     50 000 итераций случайной выборки                        │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  5. Выбор лучшего варианта (минимальное отклонение)          │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  6. Форматирование и вывод результата                        │
└──────────────────────────────────────────────────────────────┘
```

## ⚙️ Классы и API

### `models/dish.py`

| Класс | Описание |
|-------|----------|
| `Dish` | Модель блюда с полями: name, calories, proteins, fats, carbs, price, weight, recipe |
| `Meal` | Модель приёма пищи (список блюд + суммарные показатели) |
| `DietTarget` | Целевые показатели диеты |
| `DietTolerance` | Допустимые отклонения в процентах |
| `MealStructure` | Структура питания (приёмы, блюда) |

### `data/loader.py`

| Метод | Описание |
|-------|----------|
| `load_dishes_from_json(json_str)` | Загружает и валидирует JSON, возвращает список Dish или ошибку |
| `get_sample_json()` | Возвращает пример JSON для заполнения |

### `solver/diet_solver.py`

| Метод | Описание |
|-------|----------|
| `solve(target, tolerance, structure)` | Основной метод расчёта диеты |
| `_generate_possible_meals(structure)` | Генерация вариантов приёма пищи |
| `_get_deviation_score(actual, target, tolerance)` | Расчёт штрафа за отклонение |
| `_format_result(result, target, tolerance)` | Форматирование итогового JSON |

### `gui/app.py`

| Метод | Описание |
|-------|----------|
| `run_calculation()` | Запуск расчёта диеты |
| `export_result()` | Сохранение результата в файл |
| `_validate_json()` | Проверка корректности входных данных |

## ⚠️ Ограничения

| Ограничение | Описание |
|-------------|----------|
| **Количество блюд** | При >100 блюд генерация комбинаций может замедлиться |
| **Один день** | Расчёт только на 1 день (не на неделю) |
| **Эвристика** | Алгоритм не гарантирует глобальный оптимум, но находит хорошее решение |
| **Без учёта времени** | Не учитывается время приготовления блюд |
| **Без аллергенов** | Нет фильтрации по аллергенам или предпочтениям |
